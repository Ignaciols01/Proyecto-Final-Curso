2.1. Arquitectura y Tecnologías

RosterApp utiliza una arquitectura cliente-servidor moderna:

  • Frontend: SPA desarrollada con React, JavaScript, HTML5 y Tailwind CSS v4 para garantizar agilidad y responsividad.
  • Backend: Plataforma Supabase (PostgreSQL) para la gestión relacional de datos y el sistema de autenticación.

2.2. Modelo Relacional de la Base de Datos

El Modelo Entidad-Relación (MER) consta de tres entidades fundamentales para garantizar la normalización:

  • Usuarios: (id_usuario, email, password, rol). El rol discrimina entre administrador y empleado.
  • Turnos: Bloques temporales abstractos (id_turno, fecha, hora_inicio, hora_fin).
  • Asignaciones: Tabla intermedia que resuelve la relación N:M, conteniendo las claves foráneas de Usuarios y Turnos.

2.3. Requisitos Funcionales

  • Autenticación segura mediante Login.
  • CRUD completo de empleados reservado en exclusiva para el Administrador.
  • Creación de turnos y asignación múltiple de trabajadores.
  • Visualización restringida de turnos para el perfil Empleado.

2.4. Requisitos No Funcionales y Seguridad

  • Rendimiento: Carga rápida de vistas sin recargar el navegador completo (SPA).
  • Usabilidad: Enfoque Mobile First, garantizando perfecta lectura en smartphones.
  • Seguridad: Integración de Row Level Security (RLS) en Supabase impidiendo el cruce de datos privados entre empleados.
